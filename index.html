1. Introduction

CSS Grid is a powerful layout system that allows developers to create complex and responsive web designs. It provides a more efficient way to structure web pages compared to traditional methods like flexboxes.
    Advantages
  • create the complex maket with fewer lines of code
One of the main advantages of CSS Grid is its ability to create complex layouts with fewer lines of code
    Advantages
  • create the complex maket with fewer lines of code
  • control the placement of elements
It also allows you to better control the placement of elements on the page
    Disadvantage
  • compatibility with older browsers
However, one disadvantage is that it may not be supported by all older browsers. So it might requre more testing.

2-2. Column placement (fixed size)

                            .container {
                                display: grid;
                                grid-template-columns: 200px 200px;
                            }
box-1 (200px)
box-2 (200px)
box-3 (200px)
box-4 (200px)
Let's take a look how to place boxes in columns. To do that we use property 'grid-template-columns'. This property apply such sizes as pixels, percents, and another one new unit - fraction. Column number depends on how many arguments are passed to property. As we can see boxes are placed in two columns because we pass two sizes. Both columns equel fixed 200px.
2-3. Column placement (flexible size)

                            .container {
                                display: grid;
                                grid-template-columns: 1fr 1fr 2fr;
                            }
box-1 (1/3)
box-2 (1/3)
box-3 (2/3)
box-4 (1/3)
If we want to use flexible size we use units like percent and fraction. In other words if you pass size 1fr, 1fr and 2 fr, it means the first and the second boxes will be take up 1/3 part of container, and the third one - 2/3.

2-7. Areas

                            .container {
                                grid-template-rows: 1fr 1fr 1fr;
                                grid-template-areas:
                                    "first first second"
                                    "first first ."
                                    "third third third";
                            }
                            .box-1 {
                                grid-area: first;
                            }
                            .box-2 {
                                grid-area: second;
                            }
                            .box-3 {
                                grid-area: third;
                            }
box-1
box-2
box-3
There is another way to place containers. It's grid-template-areas that specifies named grid areas. We create template with names of containers and we apply these names in each cpntainer that placed within main grid container. As we can see above we named 1,2 columns and 1,2 rows as 'first', then we apply this name for 'box-1' container using property 'grid-area'. It means that container 'box-1' with property 'grid-area: first' take up 1,2 columns and 1,2 rows. Container 'box-2' take up only 3 column and 1 row, and we see that 3 column and 2 row is empty. It is because we put dot in this place accoroding to template. And the last container take up whole the third row.
